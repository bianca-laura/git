Curso de Git

Repositorio

Repositorio é o projeto dentro do Git

Configurando o Git

Criando usuario:

git config --global user.name "Bianca Laura"

criando um email:

git config --global user.email "biancalaura.rosa@gmail.com"

informar qual editor esta usando

git config --global core.editor vsCode

para resgatar as informações cadastrada usamos:

git config user.name
para o nome

git config user.email
para o email

para resgatar todas as informações cadastradas usamos

git config --list

Comandos basicos no cdm

navegando entre pastas usamos o cd

para voltar um diretorio usamos cd ../

o comando para ver os arquivos q estão dentro da pasta usamos o comando dir ou tree /f ou o ls

para criar uma pasta usamos mkdir nome-da-pasta

Criando repositorios

para iniciar o repositorio damos o seguinte comando:

git init

para ver tudo oq foi alterado ou adicionado no projeto usamos:

git status

Branch

são verções diferentes do sistema

a versão principal seria master

Comit

comit seria uma alteração no sistema

para dar o commit usamos o comando

git commit -m "Primeiro Commit"

para adicionar os arquivos ao git usamos

git add

para adicionar apenas um e

git add -A

para adicinar todos

para ver oq foi modificado usamos git log

para adicionar e comitar ao mesmo tempo vc coloca:

git -am "e o comentario sobre"

para ver qual servidor estamos usamos o

git branch

para retornar a atulização anterior usamos:

git reset --soft

volta para as atualizações anterior do comite mas com os dados ainda existentes

git reset- --mixerd

volta a atualização anterior porem os arquivos ainda nao vao estar adiocnados no git

git reset- --hard

exclui tudo das atualização

Criando outros branch

para criarmos um novo servidor usamos

git branch e o nome do novo branch

para trocar de servidor usamos o

git checkout e o nome do servidor

para verificar as alterações antes de commitar se usa o comando :

git diff

para saber somente o nome dos arquivos alterados usamos>

git diff --name-only

para saber oq foi alteração em um arquivo especifico

git diff o nome do arquivo

para desfazer as alterações usamos

git checkout HEAD -- o nome do arquivo

para adicionar o projeto no github primeiro criamos ele lá no site depois damos o seguinte comando:

git remote add origin e o caminho q ele ira dar

para verificar se foi criado damos o seguinte comando:

git remote

e depois damos o comando

] git push -u origin qual o branch q era encaminhar
